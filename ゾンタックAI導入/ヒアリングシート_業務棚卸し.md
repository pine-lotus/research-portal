# ゾンタック ヒアリングシート（業務棚卸し）

当日、これを一緒に埋める。狙いは「**どの業務に・何時間・何人かかっているか**」を数字で出し、
**時間が多く × AIで効率化しやすい**ものから着手すること。

> 棚卸しの軸を分けた（前のリストは「業務」と「IT環境」が混ざってMECEでなかった）。
> Part1＝業務（何にどれだけ時間がかかるか） ／ Part2＝環境（ITの前提） ／ Part3＝優先順位 ／ Part4＝効果測定。

## Part 3. 優先順位づけ（時間 × AI向き度）

記入後、各業務を下の図に置く。**右上（時間が多い × AI向き）から着手**する。

```
AI向き 高
  ◎        │  ★ ここから着手（最優先）
          │   例：問い合わせ返信／予約入力／売上集計
  ──────┼──────────────  月の合計時間 →
  △✕      │   （物理作業・低頻度は後回し or 対象外）
          │
AI向き 低
        時間 少 ←      → 時間 多
```

**着手の目安**：1つ目は「**時間が多くて・AI向き◎で・つらさが高い**」もの。
（仮説では `1-1 問い合わせ返信` か `1-2 予約入力` が筆頭）

## Part 4. 効果測定（before → after）

| （着手業務①） | | | | | |

> 測定のコツ：着手前に**1週間だけ実時間を記録**しておくと、before数値が正確になり、after との差＝成果がはっきり示せる。
