## 2. 環境セットアップ（Windows・あなたの手）

### 2-1. WSL2（Ubuntu）を入れる ★最初にやる

> **WSL（Windows Subsystem for Linux）とは**：Windows の中に Linux（Ubuntu）環境を作る公式機能。
> ターミナルの動きが Mac に近くなり、Claude Code が格段に安定する。

- [ ] **PowerShell を「管理者として」開く**
      スタートボタンを右クリック →「ターミナル（管理者）」または「Windows PowerShell（管理者）」を選ぶ
- [ ] 以下のコマンドを貼り付けて Enter を押す：
      ```
      wsl --install
      ```
      ※ これだけで WSL2 + Ubuntu が自動でインストールされる（数分かかる）
- [ ] **PCを再起動**する
- [ ] 再起動後、「Ubuntu」が自動で起動する → ユーザー名・パスワードを設定する
      （このパスワードは Ubuntu 用。Windows のログインとは別でOK）
- [ ] 以降の作業は **この Ubuntu のターミナル上** で行う

### 2-2. Ubuntu 上に必要なソフトを入れる

Ubuntu のターミナルに以下を順番に貼り付ける：

- [ ] **Node.js** を入れる（Claude Code の土台ソフト）
      ```
      curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
      sudo apt-get install -y nodejs
      ```
- [ ] **Git** を入れる（バージョン管理の本体）
      ```
      sudo apt-get install -y git
      ```
- [ ] **Claude Code** を入れる
      ```
      npm install -g @anthropic-ai/claude-code
      ```
- [ ] ログイン確認（`claude` と打って起動するか確認）
- [ ] 作業フォルダを作る（例：`~/zontag-ai`）＝この案件の作業場
      ```
      mkdir ~/zontag-ai
      ```

> ここから先は、Claude Code を起動して **日本語で頼めばOK。**

## 4. 「実行ボタン」を作る

WSL（Ubuntu）上では **.sh ファイル**（シェルスクリプト）が実行ファイルになる。
デスクトップに置く **.bat ファイルから WSL を呼び出す**構成にすることで、ダブルクリックで動く。

- [ ] 「ダブルクリックで次の3つを一度にやる **.bat（バッチファイル）** を作って」
      1. 最新コードを取り込む（git pull）
      2. 処理を実行する（Ubuntu 上の .sh スクリプトを呼び出す）
      3. **安全ログ**を Git に上げる（git push）
- [ ] 「安全ログは『**値（個人情報）は伏字 `***` ／ エラーの場所・種類・データの形は残す**』方針で作って」

> ※ .bat の中身は `wsl bash ~/zontag-ai/run.sh` のような1行になる。Claude Code に作ってもらえばOK。

## 5-2. エラー時のリモートサポート方法（当日仕込んでおく）

エラーが起きたときの対応を、あらかじめ3段階で決めておく：

| レベル | 状況 | 対応方法 |
|-------|------|---------|
| 軽い | エラー文が出て止まった | エラー文を LINE/メッセージで送ってもらう → こちらで診断してGitで修正 |
| 中 | 画面を見ないと分からない | Zoom 等で **Ubuntu のターミナル窓だけ**を画面共有してもらう |
| 重い | 直接操作が必要 | SSH接続（下記）で入る |

**SSH（安全なリモート端末接続）を仕込んでおく場合（できれば当日に）：**
- [ ] 「Claude Codeに頼む」：「Tailscaleを Ubuntu に入れて、SSH でリモートから入れるようにして」
      ※ Tailscale＝画面共有なしで端末だけに入れる無料ツール。個人情報が見えない。
- [ ] 自分のPCからも Tailscale を入れておく → エラー時に端末から直接操作可能になる

> **ポイント**：SSH はターミナル（文字の画面）にしか入れないため、Outlook の受信メール等は見えない。プライバシーリスクが最小。
